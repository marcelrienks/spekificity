# Setup Guide: Obsidian

## Overview

Obsidian is a markdown-based knowledge management app used in the Spekificity workflow as the storage format for the codebase graph, lessons learnt, and persistent context. The **Obsidian desktop app is optional** — you do not need it installed for the Spekificity AI workflow to function. The vault is plain markdown files on the filesystem, readable and writable by AI agents directly.

The app is useful if you want to visualise the vault graph as an interactive mind map, browse backlinks, or manually edit context notes in a rich UI.

## Install Mode

**Desktop App (Optional)** — not required for AI agent reads/writes. The vault (`vault/` directory) is plain markdown; AI agents access it via filesystem operations.

## What is an Obsidian Vault?

An Obsidian vault is a folder of `.md` files with optional YAML frontmatter. There is no binary format, no database, and no proprietary encoding. The only Obsidian-specific artefact is the `.obsidian/` config directory, which contains JSON settings and regenerates automatically when the app opens.

**This means**:
- AI agents can read and write vault files without the app running
- The vault is fully git-committable (commit `vault/` to version control)
- Opening the app after AI writes will re-index backlinks automatically

## Vault Location in Spekificity

```
your-project/
└── vault/
    ├── graph/           ← Generated by /map-codebase (Graphify output)
    │   ├── index.md     ← Master node list (AI reads this for context)
    │   ├── nodes/       ← One .md per code/doc node
    │   ├── graph.json   ← NetworkX graph (machine-readable)
    │   └── GRAPH_REPORT.md
    ├── lessons/         ← Written by /lessons-learnt skill
    └── context/         ← Maintained by AI across sessions
        ├── decisions.md
        └── patterns.md
```

## Optional App Installation (macOS/Linux)

If you want the interactive visualisation:

### macOS
1. Download from [obsidian.md/download](https://obsidian.md/download)
2. Open the `.dmg` and drag Obsidian to Applications
3. Launch Obsidian
4. Select **Open folder as vault**
5. Navigate to `your-project/vault/` and select it
6. Obsidian will index the vault and build the graph view

### Linux
1. Download the `.AppImage` from [obsidian.md/download](https://obsidian.md/download)
2. Make it executable: `chmod +x Obsidian-*.AppImage`
3. Run: `./Obsidian-*.AppImage`
4. Select **Open folder as vault** → navigate to `your-project/vault/`

## Verification

**Without the app** (confirm vault is accessible to AI):
```bash
ls vault/
# Expected: context/  graph/  lessons/
cat vault/context/decisions.md
# Expected: schema header content
```

**With the app**: Open vault in Obsidian → click the Graph View icon → nodes should appear representing your project files.

## Configuration

No configuration required for AI-only use. The `.obsidian/` directory in `vault/` is created automatically by the app on first open — commit it to version control if you want to preserve your graph layout settings.

**Gitignore consideration**: The following files regenerate on each app open and are safe to gitignore:
- `vault/.obsidian/workspace.json` — window/panel layout
- `vault/.obsidian/cache` — link index cache

These are already excluded in the project `.gitignore`.

## Troubleshooting

- **Symptom**: Obsidian app shows no graph nodes → **Fix**: Ensure you opened `vault/` as the vault root, not the project root. Re-run `/map-codebase` if `vault/graph/nodes/` is empty.
- **Symptom**: Backlinks are missing after AI writes → **Fix**: Close and reopen the vault in Obsidian to trigger re-indexing
- **Symptom**: App won't open vault (permission error on Linux) → **Fix**: `chmod -R u+rw vault/`
- **Symptom**: AI agent says vault does not exist → **Fix**: Confirm `vault/` is present at the project root; if not, run `/map-codebase` to initialise it
