# Skill Development Guide: Extending Spekificity

**Purpose**: Template and guidelines for creating new spekificity custom skills.

## Skill File Format

All spekificity skills are markdown files with a specific structure.

### Template: `spek.my-skill.md`

```markdown
# Skill: My Custom Skill

**Command**: `/spek.my-skill`  
**Namespace**: `spek.*` (spekificity custom)  
**Version**: 1.0.0  
**Author**: [Your Name]  
**Status**: active | experimental

## Description

One-sentence purpose of the skill.

## Prerequisites

- `.spekificity/config.json` must exist
- [Other prerequisites]

## Usage

```
/spek.my-skill [options]
```

### Options

- `--input VALUE` — Input parameter
- `--output PATH` — Output file path
- `--dry-run` — Preview without executing

## Invocation Examples

```
/spek.my-skill
/spek.my-skill --input data.json --output results.txt
```

## Implementation Details

[Technical details about what the skill does]

### Data Sources

- Reads: `.spekificity/config.json`, `.obsidian/graph/index.md`
- Writes: `.spekificity/` directory, project logs

### Output Format

Markdown (.md) formatted output, suitable for AI chat display.

## Error Handling

| Error | Cause | Solution |
|-------|-------|----------|
| Config not found | `.spekificity/config.json` missing | Run `spek init` first |
| Vault not initialized | Graph not built | Run `/spek.map-codebase` first |

## Testing

### Unit Test Example

```bash
# Test script
#!/bin/bash
source .spekificity/setup-scripts/logging.sh

# Run skill
/spek.my-skill --dry-run

# Verify output
if [[ $? -eq 0 ]]; then
    log_success "Skill executed successfully"
else
    log_error "Skill failed"
    exit 1
fi
```

### Integration Test

Test skill within spekificity workflow:
```bash
spek init
spek status
/spek.my-skill
```

## Namespace Compliance

✅ **Must follow**:
- File naming: `spek.<name>.md`
- Command invocation: `/spek.<name>`
- Config references: Use `spek_*` or `spek.*` prefixes
- Documentation: Reference other skills with correct namespaces

## Distribution

1. **Local skill** (project-specific):
   - Add to `.spekificity/skills/spek.my-skill.md`
   - Run `spek init` to index
   - Commit to feature branch

2. **Core skill** (spekificity repository):
   - Submit PR to spekificity project
   - Include tests and documentation
   - Skill available in future `spek update`

## Examples

### Example 1: Context-aware Analysis Skill

```markdown
# Skill: Component Impact Analysis

**Command**: `/spek.component-impact`

## Description

Analyzes how changes to a component impact dependent components.

## Usage

```
/spek.component-impact --component <name>
```

## Implementation

1. Load graph from `.obsidian/graph/index.md`
2. Find component node
3. Traverse dependency edges
4. Report downstream impacts
5. Output as markdown table

## Workflow Integration

Used in implementation phase to guide code changes.
```

### Example 2: Lesson Capture Wrapper

```markdown
# Skill: Capture Lesson Learned

**Command**: `/spek.log-learning`

## Description

Captures structured learning from current feature and stores to vault.

## Usage

```
/spek.log-learning

Prompts:
1. What was learned?
2. What worked well?
3. What could be improved?
4. Any patterns discovered?
```

## Implementation

1. Prompt for learning entries
2. Format as structured markdown
3. Store in `vault/lessons/<date>-<feature>.md`
4. Update vault index
```

## Advanced Topics

### Accessing Project Context

```bash
# Read config
source .spekificity/setup-scripts/config-handler.sh
VERSION=$(read_config ".spek_version")
BRANCH=$(read_config ".spek_platform_branch")

# Update config
update_config_string ".spek_custom_preferences.my_setting" "value"

# Validate config
validate_config || repair_config
```

### Skill Discovery and Indexing

When you add a new skill:
```bash
# Skill index is auto-generated by spek init
spek init

# Or manually regenerate:
.spekificity/setup-scripts/skill-discovery.sh generate_skill_index

# Verify skill is indexed:
grep "spek.my-skill" .spekificity/skill-index.md
```

### Logging from Skills

```bash
source .spekificity/setup-scripts/logging.sh

log_step "spekificity" "Analyzing component"
log_success "Analysis complete"
log_warning "Component has 50+ dependencies"
log_error "Component not found in graph"
log_verbose "Detailed: Found 3 related components"
```

## Contribution Guidelines

1. **Create PR** with skill markdown file
2. **Include tests** (test scenarios in doc)
3. **Document thoroughly** (usage, integration, examples)
4. **Follow namespace** (spek.* only)
5. **Update skill-index** (auto-done by spek init)
6. **Code review** before merge

## Skill Lifecycle

```
Development
  ├─ Create .spekificity/skills/spek.my-skill.md
  ├─ Test locally
  └─ Commit to feature branch

Review
  ├─ PR submitted
  ├─ Code review
  └─ Tests verified

Release
  ├─ Merge to main
  ├─ Include in next spek update
  └─ Available to all projects via `spek update`

Maintenance
  ├─ Monitor usage
  ├─ Gather feedback
  └─ Iterate on skill
```
